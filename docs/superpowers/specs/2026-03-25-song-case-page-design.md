# Song Case Page — Design Spec

**Date:** 2026-03-25
**Project:** Remix Radar (Measure of Music Hackathon)
**Status:** Approved

---

## Overview

The Song Case Page is a drill-down view centered on a single original song. It aggregates all known remixes — official and unofficial — into a single visual narrative, designed to help A&R professionals quickly assess whether an unofficial SoundCloud remix represents a DSP release opportunity. Users can save cases to a session-scoped Watch List and export a case as a PDF pitch document including a suggested split agreement.

---

## Navigation

Two entry points:

1. **Row-level button** — A "Case Page →" button on each result row in the catalog search results. Replaces the existing `TrackDetail` side panel with the full Song Case Page view.
2. **"Song Case" nav tab** — A persistent tab in the top navigation bar. Shows the most recently viewed case. Clicking from any result row both opens the case and updates this tab. If no case has been viewed yet, the tab shows an empty state.

No routing library is needed — this follows the existing `activeMode` / tab-switching pattern in `App.tsx`. The Song Case tab becomes a new mode value (e.g. `"song_case"`), and the selected song is stored in a sibling state variable.

## Page Structure

### 1. Header

- Song title (bold), artist name, label, release year, ISRC
- "Export Case PDF →" button — top right, purple filled button (matching existing design system)

### 2. Stream Comparison Chart

Horizontal bar chart. Each track (original + all remixes) gets two rows stacked:
- **SC** — SoundCloud streams
- **SP** — Spotify streams (or "Not on DSPs" in italic if absent)

Color coding encodes **platform**, not status:
- SoundCloud bars: `#ff8800` (amber orange) — used for every SC row regardless of official/unofficial status
- Spotify bars: `#1DB954` (Spotify green) — used for every SP row
- "Not on DSPs" rows: empty bar with a dashed border (`border: 1px dashed #ffffff22`) and italic "Not on DSPs" label

Status distinction (original / official / unofficial) is carried by section grouping, the `OFFICIAL` / `UNOFFICIAL` badges on artist labels, and a highlight border on standout unofficial rows. This lets users scan all SoundCloud bars together and all Spotify bars together for cross-track platform comparisons.

Unofficial remixes with notable plays get a highlighted border and background (`border: 1px solid #ff880055; background: #ff88000a`). All bars scale proportionally against the original song as the maximum.

Sections are separated by labeled dividers: `ORIGINAL SONG`, `OFFICIAL REMIXES ON DSPS (n)`, `UNOFFICIAL REMIXES ON SOUNDCLOUD (n)`.

### 3. Remix Artist Profiles

Displayed as an inline strip beneath each unofficial remix row (indented to align with the chart bars). Fields sourced from existing Chartmetric enrichment:

- Spotify monthly listeners
- SoundCloud followers
- Instagram followers
- Career Stage (e.g. "Early", "Developing")
- Momentum (e.g. "Growth", "Steady")

Official remixes do not show a profile strip (they are already on DSPs).

### 4. Auto-Generated Insight Callout

Purple-tinted callout box at the bottom of the chart. Contains:
- A one-sentence insight summarizing the key opportunity (e.g. "Justin Caruso's unofficial remix has 7.3M SoundCloud streams — more than both official remixes combined — and has never been released on any DSP.")
- Three action buttons:
  - **Recommend Release** (purple filled)
  - **Add to Watch List** (ghost button)
  - **Export Case PDF** (ghost button)

The insight text is generated by the backend (or composed client-side from structured data) using the pattern: identify the highest-stream unofficial remix, compare to official remixes, note DSP absence.

### 5. Export PDF — Contents

When "Export Case PDF" is triggered, a downloadable PDF is generated containing:

1. Song + artist metadata (title, artist, label, ISRC, release year)
2. Streaming comparison chart (screenshot or recreated vector)
3. Remix artist profiles (all unofficial remixes)
4. Revenue projections (estimated based on stream counts and industry per-stream rates)
5. Suggested split agreement template:
   - Master Rights Holder — 50%
   - Publishing / Songwriters — 25%
   - Remix Artist — 25%
   - Footer note: "Based on industry standard remix licensing splits. Subject to negotiation."
6. Recommendation (mirrors the insight callout text)

## Watch List

### Behavior
- **Session-scoped only** — no persistence between browser sessions, no server storage required.
- Clicking "Add to Watch List" saves the current Song Case (song metadata + top insight) to an in-memory list in the React state.
- A Watch List badge/count appears in the nav bar (e.g. "Watch List (3)").
- Clicking the badge opens a Watch List panel showing saved cases as cards.

### Watch List Panel
- Each card shows: song title, artist, top unofficial remix name + stream count, insight one-liner.
- Cards have a "Remove" action.
- An **"Export Watch List PDF"** button at the top generates a multi-page PDF — one page per saved case — suitable for sharing with decision makers.

## Data Requirements

All data needed for this page is already produced by the existing pipeline:

| Field | Source |
|---|---|
| Song title, artist, label, ISRC, year | `resolve_track_by_isrc` (Chartmetric) |
| Official remix streams (Spotify) | Spotify API (existing enrichment) |
| Unofficial remix streams (SoundCloud) | SoundCloud search pipeline |
| Remix artist: Spotify listeners, SC followers, Instagram | `enrich_artist` (Chartmetric) |
| Career Stage, Momentum | `enrich_artist` (Chartmetric) |

No new API calls are required. The Song Case Page is a presentation layer over data already returned in `TrackResult`.

---

## Frontend Architecture

- New mode: `"song_case"` added to `SearchMode` type in `types/index.ts`
- New state in `App.tsx`: `activeSongCase: TrackResult | null`
- New component: `SongCasePage.tsx` — receives `TrackResult`, renders the full page
- New component: `WatchList.tsx` — session state, renders panel + export
- Insight generation: compose client-side from `TrackResult` fields (no new API endpoint needed for MVP)

---

## Out of Scope (MVP)

- Persistent Watch List (server-side storage)
- Team-shared Watch Lists
- Real revenue projections (use placeholder estimates for now)
- Editable split agreement
- Clicking through to remix artist detail pages
- PDF export (jsPDF + html2canvas or similar)
